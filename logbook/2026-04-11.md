## Projects I started & worked on + things I've learned

I wanted to start establishing strong bases in Python, so I figured I'd learn multiple things at once: Docker (moreso than I do already), CI/CD, GitHub Actions and, of course, Python. Given I already have an interest in cryptography and cybersecurity, I figured I'd make a password/SSH generator API.

I genuinely could use one. I know there's a trillion already, but my usecase is a bit weirder: I generally have incredibly long passwords, but I have personal usecases for ~10 character long strings of just lowercase characters and numbers. I also do need SSH keys to be generated. Bitwarden has been a convenient tool for this but it is annoying to have to click through it. 

Yadda yadda, point is, I wanted to make it really, really reliable, not because it needs to be, but because it's cool, and I wanted to learn about CI/CD. I used FastAPI for the actual system, implemented proper test conditions, I am currently working on round-trip testing for the SSH key (i.e. testing actual signing as a test), I configured GitHub Actions to run those tests automatically.

I was really fascinated with GitHub, this stuff is genuinely a lot more useful than I thought and it makes me kinda annoyed that Codeberg doesn't really provide any of it. I will look into setting up a Woodpecker homelab for this exact purpose but the convenience and integration of GitHub Actions is hard to beat.

Anyway, regarding the actual project, I really enjoyed studying the cryptography lib from Python, and I really wanna implement more complex cryptography concepts into it. They're fun to work with, and I definitely wanna look into making my own encryption/hashing algorithm at some point. I'm aware it's stupid and you're never gonna beat AES, SHA, argon2 or whatnot, but I can try, and I'll certainly learn a lot.
